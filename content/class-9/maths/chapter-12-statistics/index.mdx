## Introduction

**Statistics** is the branch of mathematics that deals with the collection, organisation, analysis, interpretation, and presentation of data. The word "statistics" comes from the Latin word "status" meaning a political state -- because governments originally used it to keep records of population, income, and resources.

In this chapter, you will learn how to present data graphically using **bar graphs**, **histograms**, and **frequency polygons**, and how to calculate **measures of central tendency** (mean, median, and mode).

---

## Collection and Presentation of Data

**Data** are pieces of information collected for a specific purpose. Data can be:

* **Primary data:** Collected directly by the investigator for a specific purpose.
* **Secondary data:** Obtained from sources that already have the information (books, records, websites).

### Frequency Distribution

When data has many values, it is organized into a **frequency distribution table**. Data can be:

* **Ungrouped:** Each distinct value is listed with its frequency.
* **Grouped:** Values are organized into **class intervals** (e.g., 0-10, 10-20, etc.) with frequencies.

Key terms for grouped data:

* **Class interval:** A range like 10-20, where 10 is the **lower class limit** and 20 is the **upper class limit**.
* **Class size (width):** The difference between upper and lower limits (e.g., 20 - 10 = 10).
* **Class mark:** The midpoint of a class interval = (lower limit + upper limit) / 2.
* **Frequency:** The number of observations falling in a particular class interval.

---

## Graphical Representation of Data

### Bar Graphs

A **bar graph** uses rectangular bars of uniform width with equal spacing. The height of each bar represents the frequency (or value) of the category. Bar graphs are used for **discrete** or categorical data.

### Histograms

A **histogram** is similar to a bar graph but is used for **continuous class intervals**. The key differences from bar graphs are:

* There are **no gaps** between the bars (since the data is continuous).
* The width of each bar equals the class size.
* If class sizes are unequal, the height of the bar is adjusted: **height = frequency / class width** (frequency density).

> **Key idea:** When class intervals have **varying widths**, the area of the bar (not just the height) represents the frequency. The height is calculated as frequency divided by class width.

### Frequency Polygons

A **frequency polygon** is formed by:

1. Marking the **class marks** (midpoints) on the x-axis.
2. Plotting the frequencies against the class marks.
3. Joining the plotted points with straight lines.
4. Closing the polygon by connecting to the midpoints of the imaginary classes before the first and after the last class (with frequency 0).

A frequency polygon can be drawn over a histogram by joining the midpoints of the tops of the bars.

> **Key idea:** Frequency polygons are useful for comparing two distributions on the same graph, which is difficult to do with histograms.

## Measures of Central Tendency

A **measure of central tendency** gives a single value that represents the entire data set. The three common measures are **mean**, **median**, and **mode**.

### Mean (Arithmetic Average)

The **mean** is the sum of all observations divided by the number of observations.

**For ungrouped data:**

**Mean = (sum of all observations) / (number of observations)**

If the data values are x\_1, x\_2, ..., x\_n, then:

Mean (x-bar) = (x\_1 + x\_2 + ... + x\_n) / n

**For a frequency distribution:**

If values x\_1, x\_2, ..., x\_k have frequencies f\_1, f\_2, ..., f\_k, then:

Mean = (f\_1 x\_1 + f\_2 x\_2 + ... + f\_k x\_k) / (f\_1 + f\_2 + ... + f\_k) = (sum of f\_i x\_i) / (sum of f\_i)

**Example:** Find the mean of 4, 6, 7, 8, 10.

Mean = (4 + 6 + 7 + 8 + 10) / 5 = 35 / 5 = **7**

> **Key idea:** The mean uses every data value in its calculation, which means it is affected by extreme values (outliers).

### Median

The **median** is the middle value when the data is arranged in ascending (or descending) order.

* If n (number of observations) is **odd**, Median = the ((n+1)/2)th observation.
* If n is **even**, Median = average of the (n/2)th and ((n/2)+1)th observations.

**Example:** Find the median of 3, 5, 7, 9, 11.

n = 5 (odd). Median = (5+1)/2 = 3rd observation = **7**.

**Example:** Find the median of 2, 4, 6, 8.

n = 4 (even). Median = (4/2)th and (4/2+1)th values = average of 2nd and 3rd = (4+6)/2 = **5**.

> **Key idea:** The median is not affected by extreme values, making it a better measure when data has outliers.

### Mode

The **mode** is the value that occurs **most frequently** in the data set.

**Example:** Find the mode of 2, 3, 4, 5, 5, 5, 3, 2, 1.

The value 5 occurs 3 times (most frequently). Mode = **5**.

A data set can be **unimodal** (one mode), **bimodal** (two modes), **multimodal** (more than two modes), or have **no mode** (all values appear equally often).

> **Key idea:** The mode is the only measure of central tendency that can be used with **categorical** (non-numeric) data.

---

## Summary

1. **Statistics** deals with collection, organisation, analysis, and presentation of data.
2. Data can be represented using **bar graphs**, **histograms**, and **frequency polygons**.
3. Histograms are used for continuous class intervals with no gaps between bars.
4. For unequal class widths in histograms, use frequency density (frequency/class width) for height.
5. **Mean** = sum of observations / number of observations.
6. **Median** = middle value of ordered data.
7. **Mode** = most frequently occurring value.
8. Mean is affected by extreme values; median is not.
