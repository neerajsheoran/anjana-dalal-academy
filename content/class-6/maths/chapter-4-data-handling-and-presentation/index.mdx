## Why Do We Need Data?

Every day, decisions are made based on information. A doctor checks your temperature and weight before giving medicine. A shopkeeper counts how many items were sold. A teacher records marks to understand how students are doing. A government counts people to plan hospitals, schools, and roads.

This information, collected in an organised way, is called **data**.

> **Data** is a collection of facts, numbers, or observations gathered for a specific purpose.

The process of collecting, organising, and making sense of data is called **data handling** (or **statistics**).

---

## Collecting Data

Before we can analyse data, we need to **collect** it.

### Primary Data
Data you collect yourself — through observation, surveys, or experiments.

*Examples:*
- Asking 30 classmates what their favourite sport is.
- Measuring the heights of students in your class.
- Counting the number of vehicles passing a road in one hour.

### Secondary Data
Data collected by someone else that you use for your purpose.

*Examples:*
- Using census data (population count by government).
- Reading India's rainfall figures from a newspaper.
- Using BCCI statistics for cricket scores.

### How to Collect Data — A Survey

*Example situation:* A school wants to know which fruit is most popular among Class 6 students.

**Step 1:** Decide what data to collect — favourite fruit.
**Step 2:** Decide who to ask — all students in Class 6.
**Step 3:** Design the question — "What is your favourite fruit?"
**Step 4:** Collect responses — ask each student and note down the answer.
**Step 5:** Organise and present the data.

## Organising Data — Tally Marks

When data is collected, it is often messy and hard to read. **Tally marks** are a quick way to count and organise data as you collect it.

![Tally marks showing counts from 1 to 10 with every fifth stroke drawn diagonally across the previous four](tally-marks-counting.png)

**How tally marks work:**
- Draw a vertical stroke ( | ) for each item counted: |
- After 4 strokes, draw a diagonal stroke through them to make a group of 5: ||||̶
- This makes counting in fives easy and fast.

*Example — Favourite Fruits survey of 30 students:*

| Fruit | Tally | Count |
|-------|-------|-------|
| Mango | ||||̶ ||||̶ | | 11 |
| Apple | ||||̶ || | 7 |
| Banana | ||||̶ | 5 |
| Guava | ||||̶ | 5 |
| Orange | || | 2 |
| **Total** | | **30** |

> **Important:** Always check that the total of all counts equals the total number of data points collected.

## Frequency Table

Organising data into a table with categories and their counts is called a **frequency table**.

- **Frequency** = the number of times a particular value or category appears in the data.

*Example — Marks obtained by 20 students in a test (out of 10):*

Raw data: 7, 9, 6, 8, 10, 7, 8, 9, 7, 6, 8, 7, 10, 9, 8, 6, 7, 8, 9, 10

| Marks | Tally | Frequency |
|-------|-------|-----------|
| 6 | ||| | 3 |
| 7 | ||||̶ | 5 |
| 8 | ||||̶ | 5 |
| 9 | |||| | 4 |
| 10 | ||| | 3 |
| **Total** | | **20** |

Now we can easily see: most students scored either 7 or 8.

## Pictograph

A **pictograph** (or pictogram) represents data using pictures or symbols, where each picture represents a fixed number of items.

*Example — Number of mangoes sold by a fruit shop in a week:*

| Day | Mangoes sold | Pictograph |
|-----|-------------|------------|
| Monday | 30 | 🥭🥭🥭 |
| Tuesday | 50 | 🥭🥭🥭🥭🥭 |
| Wednesday | 40 | 🥭🥭🥭🥭 |
| Thursday | 20 | 🥭🥭 |
| Friday | 60 | 🥭🥭🥭🥭🥭🥭 |
| Saturday | 70 | 🥭🥭🥭🥭🥭🥭🥭 |

*(Each 🥭 = 10 mangoes)*

**Key (Scale):** Always state what each symbol represents — this is called the **key** or **scale**.

### Reading a Pictograph

*Questions you can answer from the pictograph above:*
- On which day were most mangoes sold? → **Saturday** (70)
- On which day were fewest sold? → **Thursday** (20)
- How many more were sold on Friday than Monday? → 60 − 30 = **30 more**
- What was the total sold that week? → 30+50+40+20+60+70 = **270 mangoes**

### Drawing a Pictograph

1. Choose a suitable symbol.
2. Decide the **scale** (what each symbol represents) — pick a number that divides all values neatly.
3. Draw the symbols for each category.
4. Write the key clearly.
5. Give the pictograph a **title**.

> **Tip:** If values are large (like 500, 750, 1000), choose a scale like 1 symbol = 100. If values are small, 1 symbol = 1 might work.

## Bar Graph

A **bar graph** (bar chart) represents data using rectangular bars. The **length** (or height) of each bar shows the value.

Bar graphs are more precise than pictographs because you can read exact values from the scale on the axis.

### Parts of a Bar Graph

![Labeled bar graph showing title, X-axis with categories, Y-axis with scale, bars of different heights, and gaps between bars](bar-graph-parts-labeled.png)

1. **Title** — what the graph is about.
2. **X-axis** (horizontal) — usually shows categories.
3. **Y-axis** (vertical) — shows numbers (frequency/value).
4. **Scale** — the unit interval on the Y-axis (e.g., each gridline = 10 students).
5. **Bars** — one per category, all the same width, with gaps between bars.
6. **Labels** — category names below each bar.

### Vertical Bar Graph Example

*Number of students choosing each sport in Class 6:*

| Sport | Number of students |
|-------|--------------------|
| Cricket | 14 |
| Football | 9 |
| Badminton | 11 |
| Kabaddi | 7 |
| Chess | 5 |

*(Imagine bars of heights 14, 9, 11, 7, 5 drawn on a graph with Y-axis going from 0 to 15)*

**Reading the graph:**
- Which sport is most popular? → **Cricket** (tallest bar)
- How many more chose Cricket than Kabaddi? → 14 − 7 = **7 more**
- How many students were surveyed? → 14+9+11+7+5 = **46 students**

### Horizontal Bar Graph

In a **horizontal bar graph**, bars extend sideways (left to right). The categories are on the Y-axis and values are on the X-axis. Horizontal bars are useful when category names are long.

### How to Draw a Bar Graph

**Step-by-step:**
1. Draw two perpendicular lines — the **X-axis** and **Y-axis**.
2. Write category names along the X-axis.
3. Choose a suitable **scale** for the Y-axis (mark equal intervals: 0, 5, 10, 15…).
4. For each category, draw a bar up to the correct height. Make all bars the same width.
5. Leave equal gaps between bars.
6. Label both axes and write the title.

> **Choosing the scale:** If the highest value is 60, a scale of 1 cm = 10 units works well (60 ÷ 10 = 6 cm — a manageable height).

## Mean (Average)

The **mean** (or **arithmetic mean**) is a single number that represents the "middle" or "typical" value of a dataset.

> **Mean = Sum of all values ÷ Number of values**

**Worked Example 1:** Find the mean marks of 5 students who scored: 72, 85, 68, 90, 75.

> Sum = 72 + 85 + 68 + 90 + 75 = 390
> Mean = 390 ÷ 5 = **78**

The average (mean) score is 78.

**Worked Example 2:** A shopkeeper sold the following number of samosas on 6 days: 120, 95, 110, 135, 80, 100.

> Sum = 120 + 95 + 110 + 135 + 80 + 100 = 640
> Mean = 640 ÷ 6 = **106.67 ≈ 107 samosas** per day

**Worked Example 3:** The mean temperature in Delhi for 5 days was 38°C. The temperatures on 4 days were: 36°C, 40°C, 37°C, 39°C. Find the temperature on the 5th day.

> Sum of all 5 temperatures = Mean × Number of days = 38 × 5 = 190°C
> Sum of 4 known temperatures = 36 + 40 + 37 + 39 = 152°C
> 5th day temperature = 190 − 152 = **38°C**

![Number line showing five data points with the mean marked as a balancing point in the middle](mean-balance-point.png)

### What Does the Mean Tell Us?

The mean is useful because it summarises a whole dataset with one number. But it can be misleading if one value is very different from the rest.

*Example:* In a class of 5 students, 4 students scored 20/20 and 1 student scored 0/20.
> Mean = (20+20+20+20+0) ÷ 5 = 80 ÷ 5 = **16**

The mean is 16, but this doesn't represent any actual student's score! Four students scored much higher and one scored much lower. This is why data handling involves careful interpretation, not just calculation.

## Interpreting Data — Drawing Conclusions

Data is only useful if we can draw **conclusions** from it.

*Example:* A school tracked the number of books borrowed from its library each month:

| Month | Books borrowed |
|-------|---------------|
| January | 120 |
| February | 95 |
| March | 140 |
| April | 160 |
| May | 45 |
| June | 30 |
| July | 55 |
| August | 65 |
| September | 150 |
| October | 175 |
| November | 145 |
| December | 80 |

*Conclusions we can draw:*
- Most books borrowed in **October** (175) — perhaps because exams are approaching.
- Fewest books borrowed in **June** (30) — summer holidays; students are not in school.
- There is a pattern: borrowing drops in summer and rises before exams.
- Mean borrowing per month = Total ÷ 12 = 1,260 ÷ 12 = **105 books/month**.

## Key Words to Remember

| Word | Meaning |
|------|---------|
| **Data** | Collection of facts, numbers, or observations for a purpose |
| **Tally marks** | Marks used to count in groups of 5 while collecting data |

| **Frequency table** | A table showing categories and their frequencies |
| **Pictograph** | A graph using pictures/symbols to represent data; has a key/scale |
| **Bar graph** | A graph using rectangular bars to represent data |
| **Scale** | The value each unit or symbol represents on a graph |
| **X-axis** | The horizontal axis of a graph |
| **Y-axis** | The vertical axis of a graph |
| **Mean (Average)** | Sum of all values ÷ number of values |
| **Survey** | A method of collecting data by asking questions |
| **Primary data** | Data you collect yourself |
| **Secondary data** | Data collected by someone else that you use |
